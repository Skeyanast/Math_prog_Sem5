## Практическое задание

Вариант задания №2 Ситуация 3: Продвинутый уровень:

По задание было необходимо:
1. Ситуация 1: Базовый уровень: Поле 5x5, 2 корабля по 1 клетке, вертикальное размещение.
2. Ситуация 2: Средний уровень: Поле 6x6, 2 корабля по 2 клетки, обработка полного уничтожения.
3. Ситуация 3: Поле 7x7, 3 корабля (1-2 клетки), история выстрелов.

Итого в программе реализовано:
1. (Требовалось) Возможность создания игрового поля на этапе выполнения программы с заданным пользователем размером.
2. (Требовалось) Возможность выставления кораблей пользователем с заданным размером и ориентацией.
3. Комплексная валидация выставления корабля на поле.
4. Обработка уничтожения корабля (пересчет соседних клеток).
5. (Требовалось) Обработка полного уничтожения кораблей на поле.
6. (Требовалось) История выстрелов по полю.

Задание реализовано в приложении на WinForms на языке программирования C#. В определенной степени использован паттерн MVP, предполагающий разделение приложения на цельные логические части, имеющие свои ответственности и доступ к данным. Для перехода по формам и организации приложения созданы соответствующие абстракции, представляющие базовые сущности структуры. Также для более удобной организации и масштабирования приложения применяется подход с использованием контейнера IoC.

### Окна программы

Программа содержит несколько форм:

1. Главное меню
2. Демонстрационный вариант игры (одно поле)
3. (Не реализовано) Игра (два поля, стандартные правила игры)
4. Выставление кораблей
5. Результаты игры
